***Псевдонимы (aliases, «алиасы»)*** - это удобное сокращение для часто используемых команд. Так команды становятся более короткими и запоминающимися, что экономит и снижает вероятность ошибок при вводе 

***Для создания псевдонима используем команду:*** 
- `alias имя_псевдонима='команда'`

***Пример можно сделать псевдоним ll для ls -l:*** 
- `alias ll='ls -l'`
- `ll` 
- `total 52` 
- `drwxr-xr-x 4 ubuntu ubuntu 4096 Sep 17 14:15 ./`
- `drwxr-xr-x 3 root   root   4096 Sep 16 10:32 ../`

***Еще один пример для упрощения команды sudo apt install:*** 
- `alias install='sudo apt install'`
## ***Файловые дескрипторы***

***Файловые дескрипторы*** - это числовые идентификаторы (неотрицательные целые числа). Процессы используют их для работы с файлами

***У каждого процесса есть стандартные дескрипторы, идущие первыми: 0, 1 и 2. Они используются для обозначения потоков ввода и вывода и потока ошибок***: 
- `0` - стандартный поток **ввода** 
- `1` - стандартный поток **вывода** 
- `2` - стандартный поток **ошибок и отладочной информации** 

***Обращаясь к этим дескрипторам, сис админ может управлять потоком информации процесса (используя команду ls -la результат отобразится на экране. Но с дескриптором 1 мы можем направить вывод куда угодно): 
- `ls -la 1> list.txt`

***Самый первый процесс с идентификатором 1 - /sbin/init:*** 
`/sbin/init` - один из ключевых процессов, который отвечает за инициализацию системы и управлением жизненным циклом других процессов. После загрузки ядро ОС запускает его первым

`/proc` - виртуальная ФС, содержит информацию о процессах и их дескрипторах

***Утилита для нахождения куда процессы направляют данные и откуда ждут получения:***

## ***Направления потоков***

***Направление потоков*** - это механизм управления потоками данных в процессах. Он позволяет контролировать, куда направляются данные ввода, вывода ошибок 

***Управление потоками, операторы:*** 
- `>` перенаправляет стандартный вывод в файл, перезаписывая его;
- `>>` перенаправляет стандартный вывод в файл, добавляя данные в конец файла;
- `2>` перенаправляет стандартный вывод ошибок (`stderr`) в файл;
- `&>` перенаправляет как стандартный вывод (`stdout`), так и стандартный вывод ошибок (`stderr`) в файл;
- `<` перенаправляет стандартный ввод из файла.

***Как это применять на практике*** 

`top` - утилита для вывода списках всех процессов в реальном времени на экран

***Мы получили список всех процессов. Теперь мы может сохранить его куда-нибудь для дальнейшей работы. Здесь мы и используем перенаправление вывода:*** 
- `top -b -n 1 > proc_list` (-b пакетный режим, -n 1 количество итераций)

***А теперь выведем все созданные администратором процессы: так сможем отфильтровать системные процессы и найти те, что запущены конкретным пользователем. Для этого воспользуемся перенаправлением ввода — подадим наш файл на вход утилите `grep`:***

- `$ cat proc_list | grep admin`
