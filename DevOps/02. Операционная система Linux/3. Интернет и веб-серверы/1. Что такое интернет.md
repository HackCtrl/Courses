***Всемирная паутина*** - это сеть из взаимосвязанных документов (веб-страниц), расположенных в интернете. Она задумывалась как глобальная открытая система обмена информацией, преимущественно текстовой  
## ***Интернет: технологии в основе***

#### ***URI***

***URI (Uniform Resource Identifier — «унифицированный идентификатор ресурса»)*** - это строка, которая идентифицирует абстрактный или физический ресурс. Можно сказать, что это адрес ресурса в сети

`URI = scheme ":" hier-part [ "?" query ] [ "#" fragment ]`
- `scheme` описывает способ доступа к ресурсу
- `hier-part` — расположение ресурса. Оно тоже описывается по определённым правилам, но обычно включает в себя хост, порт и путь расположения ресурса
- `query` и `fragment` — необязательные части с дополнительной информацией

https://yandex.ru/maps
- Вся строка целиком — это **URI**.
- Часть `https://yandex.ru` — уже **URL** (от англ. Uniform Resource Locator — «унифицированное местоположение ресурса»), подчинённый задаче поиска ресурса.
- `maps` — это URN (от англ. Uniform Resource Name — «унифицированное имя ресурса»). Его задача — идентификация ресурса.

***URI*** - более широкое понятие, обозначающий любой идентификатор. Его можно использовать для указания как местоположения, так и названия ресурса

***URL*** - частный случай URI, который всегда указывает конкретное местоположение ресурса в интернете 
## ***HTTP/HTTPS***

***HTTPS*** - это зашифрованный HTTP. Разница есть и в используемых ими портах по умолчанию: у HTTPS это 443 порт, а у HTTP - 80

***Где может пригодится HTTP/HTTPS:*** 
- Обеспечение пользовательского доступа в интернет через прокси-сервер 
- Поддержка веб-приложений внутри сети 
- Балансировка нагрузки на сайты компании 

*HTTP устроен как обмен текстовыми сообщениями между клиентом и сервером в формате "запрос - ответ". Эти сообщения называются HTTP-сообщениями. HTTPS - тот же HTTP, но с доп уровнем шифрования для безопасной передачи данных* 

***Структура обмена по протоколу HTTP:***
1. Запрос 
2. Заголовки 
3. Тело сообщения 
#### ***Структура запроса***

`Метод URI HTTP/Версия`
- `Метод` указывает, что мы хотим сделать — например, получить или отправить данные;
- `URI` — путь к ресурсу, с которым нужно выполнить действие;
- `HTTP/Версия` — версия протокола, согласно которой составлен запрос.

***Запрос на практике:*** 
`GET / HTTP/1.1 (получить данные ресурса /)`

***Тело сообщения*** - необязательный компонент, сами передающиеся данные. Таким образом, минимально запрос может состоять из двух строк 

`GET / HTTP/1.1 Host:ya.ru`
- `GET` — метод. Он сообщает серверу, что мы хотим получить данные.
- `/` — URI, который указывает на главную страницу сайта.
- `Host: ya.ru` — заголовок, указывающий серверу адрес сайта
#### ***Структура ответа***\

***Ответ состоит из трех частей:*** 
1. Статус ответа 
2. Заголовки ответа 
3. Тело сообщения 

`HTTP/Версия Код состояния Пояснение`
- `HTTP/Версия` — версия протокола HTTP.
- `Код состояния` — цифровой статус. Он отображает, выполнился ли запрос.
- `Пояснение` — краткое описание кода состояния для пользователя.

***Статус ответа:***
- `HTTP/1.1` — версия протокола;
- `200` — код состояния, свидетельствующий о том, что запрос выполнен успешно;
- `ОК` — описание для кода состояния `200`

***Заголовки:***
- `Expires: Tue, 19 Sep 2022 20:09:56 GMT` сообщает браузеру, когда данные на странице устареют и нужно будет заново запросить её.
- `Last-Modified: Thu, 17 Oct 2019 07:18:26 GMT` передаёт дату последнего изменения страницы.
- `Content-Length: 1256` отражает объём передаваемых данных. Эта информация помогает определить, что процесс передачи прервался.

***POST*** - служит для отправки данных 
***GET*** - для запроса 

GET используется для запроса данных с сервера, POST отправляет их для обработки, PUT создает или заменяет ресурс, DELETE удаляет
