*Отправитель шифрует сообщение с помощью своего секретного ключа, а получатель расшифровывает его с помощью другого, собственного ключа. На двух сторонах обменного канала используются разные ключи, то есть присутствует асимметрия*

![[Pasted image 20251211155957.png]]

1. У Алисы есть два ключа: **открытый (E)**, который знают все (у Боба и на схеме), и **секретный (D)**, который знает только она.
2. **Боб** шифрует исходный текст (X) **открытым ключом Алисы** и получает шифр (Y).
3. Зашифрованное сообщение (Y) отправляется по незащищенному каналу.
4. **Алиса** получает Y и расшифровывает его своим **секретным ключом (D)**, снова получая исходный текст (X).

*Здесь используется два разных ключа: **открытый — для шифрования**, его можно свободно раздавать. **Закрытый (секретный) — для расшифровки**, он хранится у получателя. Даже если злоумышленник перехватит сообщение, он не сможет его прочитать без секретного ключа. Это основа безопасной передачи данных, например, в интернете.*

*Для того, чтобы не только Боб, но и Алиса могла посылать защищенные сообщения, Боб должен аналогичным образом сгенерировать пару ключей и отправить свой открытый ключ Алисе. С его помощью Алиса сможет шифровать и отсылать сообщения Бобу, которые не сможет прочитать никто, кроме Боба.*

## ***Решение проблем***

*Для того чтобы в сети все n абонентов имели возможность не только принимать зашифрованные сообщения, но и сами посылать таковые, каждый абонент должен обладать собственной парой ключей E и D.*
